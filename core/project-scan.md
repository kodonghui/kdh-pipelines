# Project Auto-Scan (Step 0)

Universal project auto-scan protocol. Run this **BEFORE** any pipeline step. Results are cached in `project-context.yaml` at the project root.

---

## Cache Rule

If `project-context.yaml` already exists and is **< 1 hour old**, skip re-scan and use cached results.

To force a re-scan, delete `project-context.yaml` or pass `--force-scan`.

---

## Scan Steps

### 1. Package Manager Detection

Read `package.json` and detect the package manager:

```
- bun.lockb exists         → bun
- pnpm-lock.yaml exists    → pnpm
- yarn.lock exists         → yarn
- package-lock.json exists → npm
- none of the above        → npm (default)
```

Also extract from `package.json`:
- `name` → project name
- `version` → project version
- `scripts.dev` → dev server command
- `scripts.build` → build command
- `scripts.test` → test command
- `scripts.lint` → lint command

### 2. TypeScript Configuration Discovery

Find ALL `tsconfig.json` files in the project:

```
glob("**/tsconfig.json", ignore: node_modules)
```

For each discovered tsconfig:
- Record its path
- Build the tsc command: `npx tsc --noEmit -p {path}`

If monorepo detected (step 3): find the root tsconfig AND each package's tsconfig separately.

If zero tsconfig files found: `tsc_enabled = false`.

### 3. Monorepo Structure Detection

Check for monorepo tooling (in priority order):

```
- turbo.json exists              → Turborepo
- pnpm-workspace.yaml exists     → pnpm workspace
- lerna.json exists              → Lerna
- package.json has "workspaces"  → npm/yarn workspaces
- none of the above             → single-package project
```

If monorepo detected:
- List all packages/workspaces
- Record each package's name, path, and purpose
- Identify shared packages (types, UI components, utilities)

### 4. Test Runner Detection

Check for test configuration files:

```
- vitest.config.* exists                    → "npx vitest run"
- jest.config.* OR "jest" in package.json   → "npx jest"
- "bun:test" found in source files          → "bun test"
- playwright.config.* exists                → playwright_enabled = true
- cypress.config.* exists                   → cypress_enabled = true
- none found                                → test_enabled = false
```

Record:
- `test_command` — the command to run unit/integration tests
- `playwright_enabled` — boolean
- `cypress_enabled` — boolean
- `test_enabled` — boolean (false only if no test runner found at all)

### 5. BMAD Detection

Check for the BMAD method directory:

```
- _bmad/ directory exists → bmad_enabled = true
- Otherwise              → bmad_enabled = false
```

If `bmad_enabled = true`:
- Locate workflow directories (e.g., `_bmad/bmm/workflows/`)
- List available agent persona files (e.g., `_bmad/bmm/agents/*.md`)
- List available templates
- Check for critic rubric (e.g., `_bmad-output/planning-artifacts/critic-rubric.md`)

If `bmad_enabled = false`:
- Pipelines use simplified workflow (see Non-BMAD sections in pipeline docs)
- Agents are spawned with expertise descriptions only (no persona files)

### 6. UI Framework Detection

Check dependencies in `package.json` for:

```
- react-dom        → React
- vue              → Vue
- svelte           → Svelte
- @angular/core    → Angular
- next             → Next.js
- nuxt             → Nuxt
- @remix-run/react → Remix
- astro            → Astro
```

Also detect:
- Dev server command from `package.json` scripts
- Playwright config → `vrt_enabled = true/false` (visual regression testing)
- Tailwind CSS config (`tailwind.config.*`) → `tailwind_enabled = true/false`
- Other CSS frameworks (PostCSS, CSS Modules, styled-components, etc.)

### 7. Architecture Document Detection

Search for architecture documentation (check in this order):

```
- _bmad-output/planning-artifacts/architecture.md
- docs/architecture.md
- docs/ARCHITECTURE.md
- ARCHITECTURE.md (project root)
- Any file matching **/architecture*.md
```

Store the first match as `architecture_doc_path`, or `null` if none found.

### 8. Feature Specification Detection

Search for feature specification documents:

```
- _bmad-output/planning-artifacts/*feature-spec*
- docs/*feature-spec*
- docs/*features*
- Any file matching **/*feature-spec*.md
```

Store the first match as `feature_spec_path`, or `null` if none found.

### 9. PRD Detection

Search for Product Requirements Documents:

```
- _bmad-output/planning-artifacts/prd.md
- docs/prd.md
- docs/PRD.md
- Any file matching **/prd*.md
```

Store the first match as `prd_path`, or `null` if none found.

### 10. Save Results

Write all scan results to `project-context.yaml` at the project root.

## Output Format

The scan produces `project-context.yaml` with this structure:

```yaml
# Auto-generated by KDH Pipeline Step 0 — do not edit manually
# Regenerated when file is > 1 hour old or deleted

scan_timestamp: "2026-01-15T10:30:00Z"

project:
  name: "my-project"
  version: "1.0.0"
  package_manager: "bun"          # bun | pnpm | yarn | npm

scripts:
  dev: "bun run dev"
  build: "bun run build"
  test: "bun test"
  lint: "bun run lint"

typescript:
  enabled: true
  configs:
    - path: "tsconfig.json"
      command: "npx tsc --noEmit -p tsconfig.json"
    - path: "packages/app/tsconfig.json"
      command: "npx tsc --noEmit -p packages/app/tsconfig.json"

monorepo:
  enabled: true
  tool: "turborepo"               # turborepo | pnpm | lerna | npm | yarn | null
  packages:
    - name: "app"
      path: "packages/app"
      purpose: "Main frontend application"
    - name: "server"
      path: "packages/server"
      purpose: "Backend API server"
    - name: "shared"
      path: "packages/shared"
      purpose: "Shared types and utilities"

testing:
  enabled: true
  command: "bun test"
  playwright_enabled: true
  cypress_enabled: false

bmad:
  enabled: true
  workflows_dir: "_bmad/bmm/workflows"
  agents_dir: "_bmad/bmm/agents"
  templates_dir: "_bmad/bmm/templates"
  rubric_path: "_bmad-output/planning-artifacts/critic-rubric.md"

ui:
  framework: "react"              # react | vue | svelte | angular | next | nuxt | remix | astro | null
  dev_command: "bun run dev"
  tailwind_enabled: true
  vrt_enabled: true

docs:
  architecture_path: "_bmad-output/planning-artifacts/architecture.md"
  feature_spec_path: "_bmad-output/planning-artifacts/v1-feature-spec.md"
  prd_path: "_bmad-output/planning-artifacts/prd.md"
```

## Usage in Pipelines

All pipelines reference `project-context.yaml` to adapt their behavior:

- **Full Auto Pipeline**: uses `bmad.enabled` to choose BMAD vs non-BMAD workflow
- **UXUI Redesign**: uses `ui.framework` and `ui.tailwind_enabled` to determine rebuild strategy
- **Code Review**: uses `typescript.configs` for tsc validation, `testing.command` for test runs
- **E2E Testing**: uses `testing.playwright_enabled` and `ui.dev_command` for test execution
- **All Pipelines**: use `monorepo.packages` to scope operations to relevant packages
