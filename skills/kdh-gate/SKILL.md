# KDH GATE Protocol

사장님(CEO)에게 의사결정을 받아야 하는 포인트에서 사용하는 프로토콜.

## When to Use

- 다른 kdh 스킬(build, sprint, plan)에서 `[GATE]` 태그로 호출
- 직접 호출: `/kdh-gate {gate-name}`

## GATE Rules (절대 규칙)

1. **기술 용어 절대 금지** — 쉬운 말로, 비유로 설명
2. **선택지 형태** — "A할까요 B할까요?" (최대 3개)
3. **각 선택지에 장단점** — 한 줄씩
4. **한국어만** — 코드/파일명 제외 전부 한국어
5. **무한 대기** — 사장님 응답 올 때까지 절대 자동 진행 안 함
6. **`계속` 모드일 때** — GATE 스킵하고 기본 선택(A)으로 자동 진행

## GATE Inventory

### Planning Gates (기획 단계)
| # | 단계 | 질문 |
|---|------|------|
| 1 | 제품 비전 | 이 방향 맞아요? |
| 2 | 타겟 사용자 | 누구를 위한 건가요? |
| 3 | 성공 기준 | 뭘로 성공했다고 볼까요? |
| 4 | 기능 범위 | 이 기능들 넣을까요 뺄까요? |
| 5 | 디자인 방향 | 어떤 느낌으로 할까요? |
| 6 | Epic 범위 | 이만큼 만들까요? |

### Implementation Gates (구현 단계)
| # | 단계 | 질문 |
|---|------|------|
| 7 | 테마 선택 | 5개 색상 중 기본 테마 뭘로? |
| 8 | 페이지 디자인 | 이 레이아웃 맞아요? (Subframe) |
| 9 | 스프린트 확인 | 브라우저에서 확인해주세요 — OK? |

### Review Gates (리뷰 단계)
| # | 단계 | 질문 |
|---|------|------|
| 10 | 리뷰 에스컬레이션 | 이 기능을 3번이나 검토했는데 통과 못했어요. 계속 고칠까요(A), 일단 넘어갈까요(B), 처음부터 다시 만들까요(C)? |
| 11 | 자동불합격 | 심각한 품질 문제가 발견됐어요. 바로 고치고 계속할까요(A), 전체 점검할까요(B)? |

## GATE Flow

```
1. 호출 스킬이 [GATE {name}] 전송
2. AskUserQuestion으로 사장님에게 질문:
   - 한 줄 요약: "지금 뭘 결정해야 하는지"
   - 선택지 A/B/C + 각각 장단점
   - "어떻게 할까요?"
3. 사장님 응답 대기 (타임아웃 없음)
4. 응답 기록: _bmad-output/gate-decisions/{gate-name}-{date}.md
5. 결과를 호출 스킬에 전달
```

## Sprint-Verify GATE — 리뷰 요약 포함

`/kdh-gate sprint-verify` 호출 시:
```
1. Read party-logs/{sprint}-review-summary.md
2. CEO에게 보고 (한국어):
   "Sprint {N} 검토 결과입니다.
    - 전체 평균: {X.X}/10
    - 가장 높은 점수: {차원 한국어명} ({X.X})
    - 가장 낮은 점수: {차원 한국어명} ({X.X})
    - 재검토 필요했던 기능: {N}개
    
    다음 단계로 갈까요?
    A. 계속 (다음 스프린트)
    B. 안 된 기능 먼저 고치기
    C. 자세히 보여줘"
3. CEO가 C → 스토리별 D1-D6 점수를 쉬운 말로 설명
```

## `계속` Mode

`/kdh-go 계속` 으로 실행 시:
- 모든 GATE를 자동 진행 (기본 선택 A)
- 결정 기록에 `[AUTO]` 표시
- 아침에 사장님이 확인하고 변경 가능
