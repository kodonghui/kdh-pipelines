# /star-help - "뭐해야하지?"

독파모 프로젝트 상태를 읽고, 다음에 해야 할 일을 자동으로 판단한다.
**읽기만 한다. 아무것도 수정하지 않는다.**

## 사용법

```
/star-help              <- 전체 현황 + 다음 할 일 추천
/star-help gt           <- GT 현황만
/star-help docs         <- 문서/팔로업만
/star-help deadline     <- 데드라인만
```

## Phase 1: 상태 자동 감지

**전부 읽기만 한다. 아무것도 수정하지 않는다.**

인수에 따라 스캔 범위를 조절한다:
- `gt` -> 1-1 + 1-3만
- `docs` -> 1-4 + 1-5만
- `deadline` -> 1-4만
- 인수 없음 -> 전부 (1-1 ~ 1-5)

### 1-0. 프로젝트 루트 확인

```
1. 현재 작업 디렉토리에서 CLAUDE.md 존재 확인
2. 없으면 -> "독파모 프로젝트 폴더에서 실행해주세요." 후 중단
3. 있으면 -> 프로젝트 루트 경로 확정
```

### 1-1. CLAUDE.md GT 현황 테이블

```
1. CLAUDE.md 읽기
2. "현재 GT 작업 현황" 테이블 찾기
3. 각 행에서 추출:
   - 지표명 (한글)
   - 폴더 경로
   - 상태 (완료/진행중/대기)
   - 비고
4. 테이블 없으면 -> "CLAUDE.md에 GT 현황 테이블 없음" 경고 후 1-3으로 fallback

충돌 해소 (1-3과 불일치 시):
- gt/ 실제 파일 상태를 신뢰한다 (CLAUDE.md는 수동 갱신이라 지연될 수 있음)
- 불일치 발견 시 보고에 표시: "(!) CLAUDE.md와 실제 폴더 상태 불일치"
```

### 1-2. INDEX.md Active 테이블

```
1. INDEX.md 읽기
2. "Active" 테이블에서 추출:
   - 폴더 경로
   - 설명
   - 파일 수
   - 상태
3. INDEX.md 없으면 -> SKIP (경고만)
```

### 1-3. gt/ 미션 폴더 스캔

```
1. gt/ 하위 폴더 목록 (Glob "gt/*/")
   - gt/ 폴더 자체가 없으면 -> "gt/ 폴더 없음" 경고 후 SKIP
2. 각 미션 폴더에 대해:

   a) 로그 읽기:
      - Glob "gt/{미션}/log/*.md" 또는 "gt/{미션}/*-log.md"
      - 최신 로그의 마지막 섹션에서 상태 파악
      - "대상 ID" 테이블이 있으면 -> 전체 ID 수 = 목표 수

   b) 결과 카운트:
      - Glob "gt/{미션}/results/**/*.json" -> 재귀적으로 JSON 파일 수 카운트
      - 하위 폴더별로도 카운트 (true-gt/fail-gt 등)

   c) 진행률 계산:
      - 로그에 목표 수가 있으면: {결과 수}/{목표 수} 표시
      - 없으면: {결과 수}건 완료 표시

   d) 프롬프트 확인:
      - Glob "gt/{미션}/prompts/*.md"
      - 프롬프트 수 = 미션 성숙도 지표 (0이면 "뼈대만 있음")

3. 미션별 상태 판정:
   - results 폴더 비어있음 + 프롬프트 없음 -> "미시작"
   - results 폴더 비어있음 + 프롬프트 있음 -> "준비됨"
   - results 파일 있음 + 목표 미달 -> "진행중 ({N}/{M})"
   - results 파일 있음 + 목표 달성 -> "완료"
   - zip 파일 존재 -> "완료 (패키징됨)"
```

### 1-4. docs/reports/ 최신 TODO

```
1. Glob "docs/reports/*-notion-todo.md" -> 최신 파일
2. 파일 읽기
3. 체크박스 파싱:
   - "- [ ]" -> 미완료
   - "- [x]" -> 완료
4. 날짜 헤더 (### 4/8, ### 4/9 등) 기준으로 그룹핑
5. 각 날짜별 미완료 항목 추출
6. 오늘 기준:
   - 과거 날짜의 미완료 -> "지연됨"
   - 오늘 날짜의 미완료 -> "오늘 할 일"
   - 미래 날짜 -> "예정"
7. "미결 사항" 섹션 별도 추출
8. 일정 테이블 있으면 데드라인 목록 추출

notion-todo 없으면:
- docs/follow-up/ 최신 팔로업 로그에서 일정/Action Items 섹션 탐색
- 그것도 없으면 -> "데드라인 정보 없음"
```

### 1-5. docs/follow-up/ 최신 팔로업 로그

```
1. Glob "docs/follow-up/*-follow-up-log.md" -> 최신 파일
2. 파일 끝에서 역탐색 -> 마지막 "## 20XX-XX-XX" 헤더
3. 그 섹션에서 추출:
   - 핵심 결정/변경사항
   - Action Items (미완료)
   - 외부 의존 항목 (키워드: "대기", "수신", "전달", "확인 필요")
4. follow-up 로그 없으면 -> SKIP

추가로:
- docs/follow-up/ 전체 파일 목록에서 가장 최근 회의록 확인
- 회의록의 Action Items 중 미완료 항목도 수집
```

## Phase 2: 상태 보고

### 전체 모드 (인수 없음)

```
======================================
  독파모 - 현재 상태
======================================

오늘: {YYYY-MM-DD} ({요일})

GT 현황:
  {각 미션별 1줄}
  {상태 이모지}: {지표 한글명}: {상태 설명}

  이모지 규칙:
  - 완료 -> (v)
  - 진행중 -> (>)
  - 대기 -> (.)
  - 지연 -> (!)

데드라인:
  {날짜순 나열, 오늘/내일은 강조}

막힌 것:
  - {외부 의존 항목들}
  - 없으면 "없음"

미결 사항:
  - {수요일 논의 등 미확정 항목}
  - 없으면 생략

최근 작업:
  - {팔로업 로그 마지막 항목 1-2줄 요약}

======================================
```

### gt 모드

```
======================================
  독파모 - GT 현황
======================================

{미션별 상세}

미션: gt/{폴더명}/
  상태: {상태}
  결과: {N}/{M}건 ({하위 폴더별 내역})
  프롬프트: {N}종
  로그: {최신 로그 항목 1줄}

======================================
```

### docs 모드

```
======================================
  독파모 - 문서 현황
======================================

팔로업 로그:
  최신: {파일명} ({마지막 업데이트 날짜})
  미완료 Action Items: {N}건

최근 보고서:
  - {최근 3개 보고서 파일명 + 날짜}

최근 회의록:
  - {최근 3개 회의록 파일명 + 날짜}

======================================
```

### deadline 모드

```
======================================
  독파모 - 데드라인
======================================

{날짜순 전체 나열}
과거 미완료는 "지연" 표시
오늘은 "오늘" 표시
D-2 이내는 "임박" 표시

======================================
```

## Phase 3: 선택지 제시

전체 모드에서만 표시. gt/docs/deadline 모드에서는 생략.

```
다음에 뭐 할까요?

A. {가장 추천하는 다음 단계} (추천)
   - {왜 이걸 먼저 해야 하는지 한 줄}

B. {두 번째 옵션}
   - {설명}

C. {세 번째 옵션}
   - {설명}

D. 다른 거 하고 싶어 (직접 말해주세요)
```

### 선택지 결정 로직 (우선순위)

```
1. 데드라인 지연 (과거 날짜 미완료) -> 해당 작업 즉시
2. 데드라인 임박 (D-2 이내) -> 해당 GT 작업
3. 외부 의존 해소됨 (대기 항목의 데이터가 data/ 또는 gt/에 새로 존재)
   -> 해당 미션 시작 (/star-gt-mission 또는 직접 작업)
4. 진행중 GT 미완료분 -> 이어서 작업 (프롬프트 읽기)
5. 회의 D-1 -> 준비 문서 정리 (/star-docs-chain)
   (1-4 데드라인 목록에서 "회의" 키워드 + 날짜가 내일인 항목 감지)
6. 회의 직후 (1-4에서 회의 날짜가 오늘이고 노션/슬랙 미생성) -> /star-docs-chain meeting
7. 새 미션 시작 필요 -> /star-gt-mission
8. 아무것도 없으면 -> "다 했어요!"
```

### 외부 의존 해소 감지

```
CLAUDE.md 비고에 대기 패턴("~수신 후", "~확정 후", "~전달 후")이 있으면:
1. 비고에서 지표명(연결성/요청/정합성) 추출
2. data/ 에서 CLAUDE.md 마지막 수정일 이후에 생성된 파일 탐색 (mtime 비교)
3. gt/ 에서 해당 지표명이 포함된 새 폴더 탐색
4. 새 파일/폴더 발견 -> "데이터 수신된 것 같습니다: {파일명}. 확인해주세요."
5. 못 찾으면 -> "아직 대기 중"
```

## 공통 규칙

### 읽기 전용 원칙
- Phase 1~3에서 **어떤 파일도 수정/생성/삭제하지 않는다**
- 상태 보고와 추천만 한다
- 실행은 사용자가 선택지를 고른 후에만

### CLAUDE.md 준수 사항
- subcategory 순서: 요약 - 정보추출 - 추론 (보고에서도 이 순서)
- em dash 사용 금지
- 파일 참조 시 MMDD prefix 네이밍 확인

### 한국어 출력
- 모든 출력은 한국어
- 기술 용어는 최소화 (GT, subcategory 등 프로젝트 용어는 OK)

### 선택지 규칙
- 항상 A/B/C/D 4개
- A에 (추천) 표시
- D는 항상 "다른 거 하고 싶어"
- 선택지는 구체적으로: "요청 Fail GT 이어서 작업" (O), "작업 계속" (X)

## Quality Self-Check

- [ ] 5개 상태 소스를 모두 스캔했는가? (인수에 따른 범위 내)
- [ ] GT 진행률이 실제 파일 수와 일치하는가?
- [ ] 데드라인이 날짜순으로 정렬되었는가?
- [ ] 과거 미완료 항목에 "지연" 표시가 있는가?
- [ ] 선택지가 구체적인가? (어떤 GT, 어떤 문서)
- [ ] 외부 의존 항목을 빠짐없이 표시했는가?
- [ ] 파일을 수정하지 않았는가? (읽기 전용)
- [ ] 한국어로만 출력했는가?

## 유기성

### -> star-gt-mission 연결
새 미션 시작이 추천될 때: "-> /star-gt-mission {지표} {한글명}"으로 라우팅.
CLAUDE.md에 "대기" 상태이고 외부 의존이 해소된 미션이 대상.

### -> star-docs-chain 연결
회의록/보고서 파생이 필요할 때: "-> /star-docs-chain meeting" 또는 "/star-docs-chain report"로 라우팅.
최근 회의 후 노션/슬랙 미생성 시 감지.

### <-> resume-session 구분
- star-help: **프로젝트** 상태 (GT 진행률, 데드라인, 문서)
- resume-session: **세션** 상태 (이전 대화 맥락, 시도한 것, 다음 스텝)
- 둘 다 "다음에 뭐 해야 하지?"에 답하지만, 관점이 다르다
- star-help는 세션 파일을 읽지 않는다. resume-session은 GT 폴더를 스캔하지 않는다.

## 사용 가능한 star- 명령어

| 명령어 | 뭐 하는 건지 |
|--------|------------|
| `/star-help` | 지금 뭐 해야 하는지 (이거) |
| `/star-help gt` | GT 현황만 |
| `/star-help docs` | 문서 현황만 |
| `/star-help deadline` | 데드라인만 |
| `/star-gt-mission` | 새 GT 미션 시작 (폴더+프롬프트+로그) |
| `/star-docs-chain` | 회의록/보고서 -> 노션/슬랙/팔로업 자동 생성 |
| `/save-session` | 세션 저장 |
| `/resume-session` | 세션 복원 |
