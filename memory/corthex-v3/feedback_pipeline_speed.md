CEO asked "뭐가 이렇게 오래 걸리냐" (why is this taking so long?) during Stage 0.

**Why:** Each step does 4-critic review + cross-talk + fixes + re-verification + (Grade A) Devil's Advocate cycle. For a product brief this is excessive overhead.

**How to apply:** For future pipeline runs, consider:
- Grade B steps: fast-track if CEO already confirmed via GATE (reduce review depth)
- Grade C steps: execute immediately, no delay
- Cross-talk: limit to 1 round, not extended back-and-forth
- If CEO confirmed scope with "그대로 해" (no changes), minimize review scope to consistency-only
- Prioritize speed over perfection in planning stages — save thoroughness for code review
